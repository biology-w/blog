.java 中数组长度不可变。
.java 中数组元素的类型要求统一。
.数组中的元素在内存中的地址是连续的。
.所有的数组都是拿第一个数组元素的内存地址作为整个数据对象的内存地址。

### 优点：

查询/检索/获取某个下标上的元素效率极高，

#### 原因：

.每一个元素的内存地址在空间上是连续的。
.每一个元素的类型相同，所以占用的空间大小一样。

### 缺点：

数组随机增删效率低。
数组无法存储大数据量。

#### 原因：

.保证数组内元素下标顺序。
.在内存中很难找到一块大的连续内存空间。

#### 注意

对于数组中最后一个元素增删，其效率是不下标变化受影响的。
